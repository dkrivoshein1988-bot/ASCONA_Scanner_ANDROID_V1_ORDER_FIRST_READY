# ASCONA Scanner Android

Приложение для обработки возвратов в зоне Маркетплейс.

## Рабочий процесс

1. Сотрудник сканирует код заказа.
2. Приложение сохраняет номер заказа и переключается на товары.
3. Сотрудник сканирует штрихкод товара.
4. Наименование автоматически находится в локальном справочнике.
5. Позиция добавляется в текущий заказ.
6. После всех товаров сотрудник завершает заказ и переходит к следующему.

Сканирование товара заблокировано, пока не получен код заказа. Диалог выбора поля после сканирования не используется.

## Возможности версии 1.0

- полноэкранный сканер на CameraX/Google ML Kit;
- автоматический зум и фокус по касанию;
- EAN-13, EAN-8, Code 128, Code 39/93, ITF, UPC, QR, Data Matrix и PDF417;
- локальный справочник на SQLite, работает без интернета;
- 11 049 пар `штрихкод + наименование` из `справочник.xlsx`;
- выбор товара для семи неоднозначных штрихкодов;
- ручной ввод и поддержка сканера ТСД/Bluetooth HID;
- поиск товара по названию;
- журнал позиций, исправление и удаление;
- состояние товара, сотрудник, смена и маркетплейс;
- экспорт CSV с кодом заказа, ШК и наименованием.

## Сборка APK через GitHub

1. Загрузите содержимое проекта в корень репозитория.
2. Убедитесь, что файл `.github/workflows/build_android.yml` присутствует.
3. Откройте вкладку `Actions`.
4. Выберите `Build Android APK`.
5. Нажмите `Run workflow`.
6. После зелёного результата откройте запуск и скачайте артефакт `ascona-scanner-android-v1`.
7. Распакуйте артефакт и установите `app-release.apk` на Android.

Workflow самостоятельно создаёт Android-платформу, добавляет разрешение камеры, выполняет `flutter analyze`, тесты и release-сборку.

## Обновление справочника

Исходный Excel должен содержать:

- колонка A — штрихкод;
- колонка B — наименование товара;
- первая строка — заголовки.

В PowerShell из папки проекта:

```powershell
powershell -ExecutionPolicy Bypass -File .\tool\build_product_catalog.ps1 `
  -SourceXlsx "C:\путь\к\справочник.xlsx"
```

После обновления загрузите новый `assets/product_catalog.json` в репозиторий и запустите сборку.

## Структура

- `lib/screens/returns_home_page.dart` — рабочий процесс и интерфейс;
- `lib/screens/scanner_page.dart` — изолированный полноэкранный сканер;
- `lib/services/product_catalog.dart` — SQLite и поиск товаров;
- `lib/services/return_storage.dart` — локальная история и настройки;
- `assets/product_catalog.json` — подготовленный каталог товаров;
- `tool/build_product_catalog.ps1` — генератор каталога из Excel;
- `test/barcode_utils_test.dart` — проверки обработки кодов.
