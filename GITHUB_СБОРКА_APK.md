# Сборка APK в GitHub

## Загрузка проекта

Рекомендуемый способ — распаковать подготовленный архив и загрузить все файлы в пустой репозиторий. В корне репозитория должны находиться `lib`, `assets`, `test`, `tool`, `pubspec.yaml` и `README.md`.

GitHub иногда не загружает скрытую папку `.github` перетаскиванием. В этом случае:

1. Нажмите `Add file` → `Create new file`.
2. В поле имени введите `.github/workflows/build_android.yml`.
3. Откройте корневой файл `WORKFLOW_BUILD_ANDROID.yml` на ноутбуке.
4. Вставьте его содержимое в созданный файл.
5. Нажмите `Commit changes`.

## Запуск

1. Откройте `Actions`.
2. Слева выберите `Build Android APK`.
3. Нажмите `Run workflow` → `Run workflow`.
4. Дождитесь зелёного статуса.
5. Откройте завершённый запуск.
6. Внизу страницы скачайте `ascona-scanner-android-v1`.
7. Распакуйте ZIP-артефакт и установите `app-release.apk`.

Сборка выполняет анализ кода и тесты до создания APK. Если один из этих этапов не проходит, APK не публикуется, а точная ошибка остаётся в журнале Actions.
